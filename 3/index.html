jsx

jsx 是一種 JavaScript 的擴充語法

看起來像是 XML

Babel 有支援 jsx 的編譯

用小寫來表示 html element，像是：

<div className="foo" />
或是 
<div></div>
用大寫來表示 React Component： 
<Hello text="word" />
跟 XML 一樣 tag 一定要 close (/)
背後其實是用 `React.createElement` 去作轉換 
// Input (JSX):
var app = <Nav color="blue" />;
// Output (JS):
var app = React.createElement(Nav, {color:"blue"});

// Input (JSX):
var app = <Nav color="blue"><Profile>click</Profile></Nav>;
// Output (JS):
var app = React.createElement(
  Nav,
  {color:"blue"},
  React.createElement(Profile, null, "click")
);

Component render 常見錯誤

錯誤範例：

回傳的根 element 只能有一個

class Hello extends React.Component {
  render() {
    // 不能回傳三個並列的
    return (
      <div>Hello World</div>
      <div>Hello World</div>
      <div>Hello World</div>
    );
  }
}

超過一個必須用陣列形式：

class Hello extends React.Component {
  render() {
    // 陣列則 ok
    return [
      <div>Hello World</div>,
      <div>Hello World</div>,
      <div>Hello World</div>
    ];
  }
}

把並行元素包進一個共同根 element 是比較普遍的做法：

class Hello extends React.Component {
  render() {
    // 包進 div
    return (
      <div>
        <div>Hello World</div>
        <div>Hello World</div>
        <div>Hello World</div>
      </div>
    );
  }
}

state

通常是比較高階的 component 才會有

有 state 的 component 被稱為 stateful component

class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = {count: props.initialCount};
  }
  tick() {
  	// 只用 setState 去改變 state
  	// 直接修改 this.state 是無法自動觸發變更的
    this.setState({count: this.state.count + 1});
  }
  render() {
    return (
      <div onClick={this.tick.bind(this)}>
        Clicks: {this.state.count}
      </div>
    );
  }
}
Counter.defaultProps = { initialCount: 0 };
Counter.propTypes = { initialCount: React.PropTypes.number };

Controlled Components

// 沒有 value 就會是 Uncontrolled Component
// 跟一般一樣
// 但沒有辦法用 react 去控制 value
render: function() {
  return <input type="text" />;
}

// Controlled Component
// 指定了值，那就無法改變它
render: function() {
  return <input type="text" value="Hello!" />;
}

通常的 Controlled Component 手法

getInitialState: function() {
  return {value: 'Hello!'};
},
handleChange: function(event) {
  this.setState({value: event.target.value});
},
render: function() {
  var value = this.state.value;
  return <input type="text" value={value} onChange={this.handleChange} />;
}

// 像 twitter 一樣限制 140 個字
handleChange: function(event) {
  this.setState({value: event.target.value.substr(0, 140)});
}

Event System

React 使用 SyntheticEvent

  • 在 nativeEvent 上封裝來達成跨平台

Event pooling：會把 event 重複使用，要很小心

class Demo extends React.Component {
  handleClick(event) {
    console.log(event); // => nullified object.
    console.log(event.type); // => "click"
    var eventType = event.type; // => "click"

    setTimeout(function() {
      console.log(event.type); // => null
      console.log(eventType); // => "click"
    }, 0);

    this.setState({clickEvent: event}); // Won't work. this.state.clickEvent will only contain null values.
    this.setState({eventType: event.type}); // You can still export event properties.
  }
  render() {
  	return <button onClick={this.handleClick.bind(this)}>
  }
}
